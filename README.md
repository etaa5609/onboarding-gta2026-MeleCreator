## S01 — Fiche d'opportunite agentique

Votre premier livrable est une **fiche d'opportunite agentique** basee sur un des trois cas vus en classe (Klarna, GitHub Copilot Enterprise ou Morgan Stanley).

Un gabarit pre-rempli avec les 6 champs est disponible dans :
`portfolio/L1_reflexion_role_specialise.md`

1. Ouvrez le fichier `.md`, remplissez les 6 champs
2. Exportez en PDF (ou redigez directement un PDF)
3. Deposez `L1_reflexion_role_specialise.pdf` dans `portfolio/`
4. Mettez a jour `ai-usage.md`

## Comment soumettre un livrable

Vous n'avez **pas besoin** de connaître Git. Tout se fait dans le navigateur.

### Étape 1 — Préparez votre fichier PDF

Nommez votre fichier **exactement** comme indiqué dans le tableau ci-dessous.
Un nom incorrect = livrable introuvable lors de la correction.

**Jalons lourds (1,5 % chacun, 9 % total) :**

| Jalon | Séance | Nom de fichier exact |
|-------|--------|----------------------|
| M1 | S02 | `M1_grille_selection_agents.pdf` |
| M2 | S04 | `M2_memo_finserv.pdf` |
| M3 | S07 | `M3_rapport_evaluation.pdf` |
| M4 | S09 | `M4_business_case.pdf` |
| M5 | S12 | `M5_audit_ethique.pdf` |
| M6 | S14 | `M6_feuille_route_adoption.pdf` |

**Exercices légers (1 % chacun, 6 % total) :**

| Exercice | Séance | Nom de fichier exact |
|----------|--------|----------------------|
| L1 | S01 | `L1_reflexion_role_specialise.pdf` |
| L3 | S03 | `L3_risques_cycle_vie.pdf` |
| L6 | S06 | `L6_reflexion_strategie.pdf` |
| L8 | S08 | `L8_mini_pipeline.pdf` |
| L11 | S11 | `L11_metriques_agent.pdf` |
| L13 | S13 | `L13_choix_hitl.pdf` |

### Étape 2 — Déposez le PDF dans le dossier `portfolio/`

1. Ouvrez ce dépôt dans votre navigateur sur github.com
2. Cliquez sur le dossier **`portfolio`**
3. Cliquez sur **Add file → Upload files** (en haut à droite)
4. Glissez-déposez votre PDF ou cliquez pour le sélectionner
5. Dans la section **Commit changes**, laissez le message par défaut ou écrivez : `M3 — Canevas de cas d'usage` (exemple)
6. Cliquez **Commit changes**

C'est tout. La date et l'heure du dépôt sont enregistrées automatiquement.

> **Si vous mettez à jour un livrable** : re-déposez le fichier avec le même nom.
> L'ancienne version est conservée dans l'historique — vous ne perdez rien.

### Étape 3 — Mettez à jour ai-usage.md

1. Cliquez sur **`ai-usage.md`** à la racine du dépôt
2. Cliquez sur l'icône crayon ✏️ (Edit this file)
3. Remplissez les 5 sections pour ce milestone
4. Cliquez **Commit changes**

`ai-usage.md` est **obligatoire à chaque milestone**, même si vous n'avez utilisé aucun outil IA.
Écrivez simplement « Aucun » dans chaque section et signez.

## Politique d'usage de l'IA

Les outils IA sont permis **lorsqu'explicitement indiqué** dans le guide de studio.

Toute utilisation doit être déclarée dans `ai-usage.md`. Vous êtes responsable de
l'exactitude, du jugement et de la vérification de tout contenu produit — outil IA ou non.

Gabarit de déclaration disponible ici : `ai-usage.md` (ce fichier est pré-rempli dans votre dépôt).
